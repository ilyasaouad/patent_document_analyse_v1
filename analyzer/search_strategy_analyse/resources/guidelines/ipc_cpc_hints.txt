IPC / CPC CLASSIFICATION — COMPLETE CLASS-LEVEL REFERENCE
==========================================================

HOW TO USE
----------
This file lists ALL IPC/CPC sections and classes (2-digit level, e.g. A01, G06).
The LLM should:
  1. Use this reference to identify the most relevant CLASS(es) for the patent.
  2. The pipeline will then call the EPO Linked Open Data API to fetch the full
     subclass/group hierarchy for each identified class, enabling refinement
     to the most specific subgroup.

API endpoint (no auth required):
  https://data.epo.org/linked-data/def/cpc/{SYMBOL}.json
  (use '-' instead of '/' — e.g. G06N3-00 for G06N 3/00)

═══════════════════════════════════════════════════════════════════════════
NOTES FOR THE LLM
═══════════════════════════════════════════════════════════════════════════
- Identify 1–5 most relevant CLASSES (e.g. G06, H04, A61) from this list.
- The pipeline will use the EPO API to fetch the full subclass tree for each.
- CPC codes are more granular than IPC — prefer CPC subgroups in EPO searches.
- Many inventions span multiple sections (e.g. a medical AI device → A61 + G06 + G16).
- For hybrid analog-digital inventions, check BOTH G06 and H03.
- Y-section codes (CPC-only, not in IPC) are tagging codes for cross-cutting
  technologies (e.g. Y02 = climate change mitigation, Y04 = smart grids).
  These can be added alongside main classification codes.
