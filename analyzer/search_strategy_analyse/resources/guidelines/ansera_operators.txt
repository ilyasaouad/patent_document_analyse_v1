ANSERA (PQL) – Proximity & Operator Rules

==========================================================================
CRITICAL: TRUNCATION vs. MULTI-WORD CONCEPTS
==========================================================================
Do not use "*" in truncation, use "+" for more than 1 trailing characters or "?" for only 1 trailing character.

The "+" operator is ONLY a character-level wildcard on a SINGLE word.
It means "0 or more trailing characters". It does NOT join words together.

WRONG (NEVER DO THIS):
  analog+digital+comput+
  → This does NOT mean "analog digital computer"
  → PQL reads it as ONE word: "analogdigitalcomput..." (nonsense)

CORRECT (ALWAYS DO THIS):
  Use proximity operators (nW or nD) between separate words.
  Use comma (,) for OR alternatives.

  (analog, digital) 2D comput+
  → Finds "analog" or "digital" within 2 words of "computer/computing/..."

  analog 2W digital 2W comput+
  → Finds "analog" then "digital" then "computer/..." in that order, each within 2 words

MORE CORRECT EXAMPLES:
  WRONG:  analog+digital+comput+ OR hybrid+comput+
  RIGHT:  (analog, digital, hybrid) 2D comput+

  WRONG:  field+programmable+gate+array
  RIGHT:  field 2W programm+ 2W gate 2W array+

  WRONG:  machine+learning+model+
  RIGHT:  machine 2W learn+ 2W model+

  WRONG:  heat+exchang+ OR thermal+manag+
  RIGHT:  (heat 2D exchang+), (thermal 2D manag+)

SHORTHAND OPERATORS:
  ,       = OR        →  apple, banana  means  apple OR banana
  space   = 0W        →  yellow banana  means  yellow 0W banana (adjacent, ordered)
  _       = ab or a b →  white_house    matches whitehouse or white house

==========================================================================
TRUNCATION OPERATORS (character-level, SINGLE word only)
==========================================================================
  +  → 0 or more characters    comput+  → computer, computing, computation
  ?  → 0 or 1 character        colo?r   → color, colour
  #  → exactly 1 character     gr#y     → gray, grey

  These apply to ONE WORD only. Never chain them to join multiple words.

  The + can also appear at the START of a word for left-truncation:
  +motor  → finds electromotor, servomotor

  Combined left+right truncation: +conduct+ → semiconductor, superconductor, etc.

==========================================================================
PROXIMITY OPERATORS
==========================================================================

nW — Ordered proximity: a nW b
  a appears before b with at most n unmatched words between them.
  Example: yellow W banana → matches "yellow banana" (adjacent, ordered)

nD — Unordered proximity: a nD b
  a and b appear within n words of each other, in ANY order.
  Example: yellow D banana → matches "yellow banana" AND "banana yellow"

nS — Sentence proximity: a nS b
  a and b within n sentences of each other.

==========================================================================
FRAMING OPERATORS (BEST PRACTICE for complex queries)
==========================================================================

nUG(term1, term2, ..., termN) — Unordered group
  Matches word sequences of at most n words containing ALL terms in ANY order.

nOG(term1, term2, ..., termN) — Ordered group
  Same but terms must appear in the GIVEN order.

OPERATOR EQUIVALENCE:
  0D ≈ 2UG
  1D ≈ 3UG
  2D ≈ 4UG
  3D ≈ 5UG

Example:
  5UG(Door, (lock, latch, bolt), installation)
  → Finds all three concepts within a 5-word window, any order

  Advantages over chained D/W:
  - Treats all terms as one group
  - Avoids first-match evaluation issues
  - More robust recall

==========================================================================
COMMON FAILURE WITH CHAINED PROXIMITY + OR
==========================================================================

Query:   Door 2D (lock, actuator) D installation
Text:    Door lock actuator installation
Problem: "lock" matches first → "actuator" adds distance → "installation" is now too far → NO HIT

KEY RULE: When chaining proximity operators, outer distance ≥ inner distance
  Door 2D (lock, actuator) 2D installation  ← works

BEST FIX: Use a single framing operator instead
  5UG(Door, (lock, actuator), installation)  ← always works

ALTERNATIVE: Expand synonyms manually
  Door 2D lock D installation, Door 2D actuator D installation

==========================================================================
EVALUATION ORDER
==========================================================================
Queries are parsed LEFT → RIGHT:
  heart 1D rate 3D frequency = (heart 1D rate) 3D frequency
Use parentheses to control logic explicitly.

==========================================================================
ANTI-PROXIMITY OPERATORS
==========================================================================
!nW  — Match a but NOT when followed by b within n words
  san !W francisco → finds "san" but not "san francisco"

nov  — No overlapping
  san nov (san francisco) → finds "san" alone

==========================================================================
BOOLEAN OPERATORS
==========================================================================
AND  — Both terms required
OR   — Either term (shorthand: comma ,)
NOT  — Exclude term
  a NOT b = a AND NOT b

==========================================================================
GROUPING AND ESCAPING
==========================================================================
( )   — Grouping: (green, red) 3D apple+
" "   — Escape PQL operators: "3d" graphics card  (searches literal "3d")
[ ]   — Alternatives within a word: dat[a,um,] → data, datum, dat
&     — Literal (case-sensitive): &Cu, &CAN

==========================================================================
BEST PRACTICES SUMMARY
==========================================================================
1. NEVER use + to join multiple words. Use nW/nD proximity instead.
2. Use comma (,) for OR alternatives, NOT the word "OR" in proximity groups.
3. Prefer UG/OG framing operators over chained D/W for complex queries.
4. Use generous proximity distances (real text has reference signs, OCR artifacts).
5. Use parentheses for clarity.
6. Keep queries simple and readable.

RULE OF THUMB:
  If your query has: proximity + (synonyms) + proximity
  → Replace with a single UG/OG framing operator.

EXAMPLE SEARCH STRATEGY MARKS:

Mark A — analog/digital computing hardware:
  WRONG:  analog+digital+comput+ 3W (accelerator, card)
  RIGHT:  (analog, digital, hybrid) 2D comput+ 3D (accelerator, card)
  BEST:   5UG((analog, digital, hybrid), comput+, (accelerator, card, processor+))

Mark B — heat management:
  WRONG:  heat+exchang+ OR thermal+manag+
  RIGHT:  (heat 2D exchang+), (thermal 2D manag+), (cool+ 2D system+)
